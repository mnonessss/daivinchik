# Dating Bot

Telegram-бот знакомств с backend на FastAPI, очередями задач, кэшем и хранилищем фотографий.

## Архитектура

Проект построен по сервисной модели:

- **Telegram Users** — конечные пользователи в Telegram.
- **Telegram Bot** — точка входа, принимает команды и действия пользователей.
- **FastAPI Backend** — центральный API и бизнес-логика:
  - `Users`
  - `Profiles`
  - `Interactions`
  - `Ranking`
- **PostgreSQL** — основное хранилище данных.
- **Redis Feed Cache** — быстрый кэш ленты.
- **RabbitMQ Event Queue** — очередь событий для асинхронной обработки.
- **Celery Workers** — фоновые задачи
- **MinIO (S3 Storage)** — хранение фотографий профилей.


## Модули backend

### Users

Отвечает за регистрацию и храенение данных пользователя:

- создание пользователя при первом входе;
- хранение `telegram_id`, активности, реферального источника;
- поддержка базового статуса пользователя.

### Profiles

Отвечает за CRUD анкеты пользователя:

- персональные данные (имя, возраст, пол, город);
- биография;
- счетчик фото;
- предпочтения поиска (возраст/город).

### Interactions

Отвечает за действия между пользователями:

- лайк/дизлайк/скип;

### Ranking

Отвечает за ранжирование анкет:

- хранение базового и поведенческого рейтинга;
- расчет итогового `final_score`;
- регулярный пересчет через фоновые задачи.
